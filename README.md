# SimpleEMR

**Offline-first electronic medical records for rural and low-resource clinics.**

SimpleEMR is a lightweight EMR that runs entirely in the browser — no server, no account, no internet connection required. Patient data lives in the browser's own database (IndexedDB), so a clinic can chart on any device, even fully offline.

Part of the projects at [annasrahman.com](https://annasrahman.com).

## Features

- **Patient charts** — demographics, encounter timeline, and medication lists in one view
- **Encounters** — create and browse visit notes on a chronological timeline
- **Fast patient search** — find patients by first or last name as you type
- **Offline-first** — all data is stored locally via IndexedDB (Dexie); nothing leaves the device
- **Import / export** — back up or transfer the entire database as a file (`dexie-export-import`)
- **Demo data** — seeds itself with synthetic patients so you can explore immediately

> **Note on data:** every patient in this repository and in the demo is synthetic, generated by the seed script. No real patient information is present anywhere in the code or its history.

## Tech stack

- [React 19](https://react.dev) + TypeScript
- [Vite](https://vite.dev) for build and dev server
- [Dexie](https://dexie.org) (IndexedDB) for local-first storage
- [Tailwind CSS](https://tailwindcss.com)

## Getting started

```bash
npm install
npm run dev
```

Then open the printed local URL. The app seeds a demo database on first launch.

Other scripts:

```bash
npm run build     # type-check and build for production
npm run lint      # eslint
npm run preview   # preview the production build
```
